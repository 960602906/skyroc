# 前端接口对接与页面完善清单

> 用途：这是 SkyRoc 前端的唯一续接任务序列。每次“继续前端对接”或“完善界面”时，先读取本文件和 [`前端开发进度.md`](./前端开发进度.md)，只领取第一个不是「已完成」或「阻塞」的任务。
>
> 基线核对：2026-07-20；菜单叶子页 63 个，其中已实现 20 个、`PagePlaceholder` 占位页 41 个、尚无页面文件 2 个。后端接口一律以控制器和运行时 OpenAPI 为准；`.cursor/api-diff-report.json` 只能辅助发现遗漏，不能替代后端契约。

## 1. 执行规则

1. 顺序以任务 ID 为准，**不得跳项、不得并行领取多个菜单页面**。当前唯一任务见进度文档。
2. 一个任务必须完成该页需要的 `types → urls → api → hooks/调用页 → 路由/i18n/权限 → 验收`；仅新增 API 函数或仅替换占位页都不能完成任务。
3. 对直接映射的后台 DTO 先收紧 `Api.*` 类型，再调用 `src/service/request`。业务代码不得读取 `response.data`；列表使用 `useTable`，全页详情/编辑首屏使用路由 `loader`。
4. 后端业务失败的 HTTP 状态仍为 200，前端只按响应体 `code` 判断；按钮以接口权限和动态路由返回的按钮权限共同控制。
5. 任务通过验收后：本表状态改为「已完成」、更新进度文档的统计/断点/测试基线，并把下一条未完成项设为唯一当前任务。

### 状态和服务层标记

| 标记 | 含义 |
| --- | --- |
| 已完成 | 页面不是占位，实现、路由和主要交互已接入；后续仅在接口变化时回归。 |
| 待页面 | 页面文件存在但仍为 `PagePlaceholder`；服务层可能已存在，仍须按实际 DTO 复核。 |
| 待接口 | 页面为占位且接口层缺少该页必需契约或函数；先补服务层。 |
| 待路由 | 后端和服务层已存在，但页面目录/路由尚未创建。 |
| 阻塞 | 需要后端新增或澄清契约时使用；记录证据后不得继续实现该项。 |

服务层列中的「已有」仅表示对应 `types/urls/api` 文件或函数存在，不代表 `Record<string, unknown>` 等宽泛类型已满足页面所需；页面实现时必须按后端 DTO 收紧。

接口列先写 Controller 的完整 `GET/POST /api/...` 基路径；同一单元格后续以 `/` 开头的短路径均严格追加到该 Controller 的 `[Route("api/...")]`，例如 `OrdersController` 的 `GET /{id}` 即 `GET /api/orders/{id}`。这只是不重复书写公共前缀，HTTP 方法和完整路由均以对应 Controller 为准。

### 通用验收编号

| 编号 | 可执行验收 |
| --- | --- |
| A0 | `pnpm typecheck` 通过；接口使用 `request`、无 `any` 或 `response.data` 解构；页面文案、枚举和日期展示符合前端规则。 |
| A1 | 列表具备加载、空态、分页、筛选/重置、刷新和无权限按钮收口；分页查询使用 `useTable`。 |
| A2 | 新增/编辑/详情按页面类型实现；全页表单/详情使用 loader、分区卡片和 `useCloseTabAndNavigate`，保存后刷新或回列表。 |
| A3 | 审核、作废、签收等状态动作有权限控制、确认/必填备注、成功提示和列表刷新；非法状态由后端错误提示呈现。 |
| A4 | 看板按统计周期查询，金额/数量/日期口径与接口说明一致；图表、排行或状态汇总具备空数据态。 |
| A5 | 只读报表有查询条件、分页或全量结果的适当展示、金额/日期格式化；不得在报表页写入业务数据。 |
| A6 | 审计/查询页为只读；筛选字段、脱敏字段和 UTC 时间均按接口约束展示。 |
| A7 | 上传/下载使用 `multipart/form-data` 或二进制响应处理；显示后端限制和失败提示，不伪造文件管理列表。 |
