## 每项完成后的更新要求

1. 在任务清单中把当前行改为「已完成」，补充实际新增的页面、接口或测试证据；不能只改变进度数字。
2. 更新本文件的当前唯一任务、各状态统计、最近完成项和验证基线。
3. 运行清单第 3 节的静态核对、`pnpm typecheck`、`git diff --check`；页面改动再以 `pnpm dev` 手动走通本任务主流程。
4. 如后端缺少完成页面所必需的契约，将任务改为「阻塞」，记录 Controller/接口证据和所需决定；不得跳过到下一项。
