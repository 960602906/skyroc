## P3 — 页面逻辑与路由/类型

### 11. 编辑页业务逻辑内联，且两种加载策略并存

- **现状：**
  - `src/pages/(base)/master/customer/operate/[id].tsx:20-47`：内联 `useEffect` 做 `fetchGetCustomerDetail` + `form.setFieldsValue`，`handleSubmit` 直接调 `fetchUpdateCustomer`。
  - `src/pages/(base)/master/goods/operate/[id].tsx:6-52`：同类内联，另用 React Router `loader` 调 `fetchGetGoodsDetail`/`redirect`。
- **问题：** 两个相邻模块用**不同**加载策略（`useEffect` 加载 vs 路由 `loader` 加载），编辑流程手工复制而非沉淀到 feature hook。（注：列表页整体很「薄」，已良好委托给 `useTable`/`CrudPageLayout`/`toggleEntityStatus`——此问题仅限编辑页。）
- **建议：** 统一编辑页数据加载策略（建议采用路由 `loader`，项目已有 `route-loader-data` 文档与实践），或抽象 `useEntityEditForm` 类 feature hook 复用「加载详情→回填→提交」。

### 12. 路由层死代码与可疑写法 — ✅ 已处理

- **已做：** 删除 `filterAuthRoutesByDynamic`、`VITE_CONSTANT_ROUTE_MODE`；`allRoutes` 改为直接使用 `customRoutes` 数组；lazy 返回去掉 `any`（以 `LazyRouteResult` 断言满足 RR 类型）。

### 13. 51 个手写 `Api.*` 类型文件——最大的类型维护面

- **现状：** `src/service/types/*.d.ts` 51 个文件，按 `Api.<Module>` 命名空间**手写**，随后端 DTO 手工同步。项目已有活跃 Apifox 项目。
- **问题：** 手工维护易漂移（参见第 7 项占位类型），是「应生成而未生成」最明确的信号。
- **建议：** 评估从 Apifox/OpenAPI 生成 `Api.*` 类型（如 `openapi-typescript`）。属较大投入，见第 15 项决策。

### 14. 其它小项 — 🟡 部分完成

- **已做：** 水印改用 `VITE_APP_TITLE`；`CLAUDE` 补全 `@sa/fetch`/`color`；删除菜单 `QueryForm` 死代码与孤立 `query-form.tsx`；删除模板 `CHANGELOG*.md`。

## 值得保留的良好实践（勿动）

- **包依赖方向干净：** `packages/**` 对 `@/`、`~/`、`import.meta.env.VITE_` 搜索**零命中**——workspace 包不反向依赖应用代码/环境，env 解析正确留在 `src/`（`utils/service.ts`、`service/request/`），`@sa/axios` 保持 env 无关。
- **api 层纯净：** 46 个 api 文件无一处 `response.data` 泄露、无业务逻辑，`transformBackendResponse` 统一在 `request` 实例处理。
- **存储纪律：** 无裸 `localStorage`/`sessionStorage` 调用，全部走 `localStg`/`sessionStg`（`src/utils/storage.ts`）。
- **列表页很薄：** CRUD 列表页良好委托给 `useTable`/`useTableOperate`/`CrudPageLayout`/`toggleEntityStatus`/`renderEnableStatus`；页面局部纯函数已抽到同目录 `*-utils.ts`/`shared.ts`（如 `customer-form-utils.ts`、`menus/modules/shared.ts`）。
- **`RootState` 由 `ReturnType<typeof rootReducer>` 推断**（`src/store/index.ts:15`），无手写。
- **`components/`（跨切面）与 `crud`/`table`/`theme`（feature 局部）边界清晰**，无重复。
- **懒加载权限：** `patchRoutesOnNavigation` 首次鉴权导航时注入路由（`features/router/router.ts:46-55`），单次 `isAlreadyPatch` 守卫。
- 严格 TS（`strict`/`isolatedModules`）；路由 lazy 已消除原 2 处 `any`；无 `TODO`/`FIXME`/`HACK` 债务标记。
