### 第一轮修复：详情页 i18n 问题

#### 1. 详情页组件导入缺失
**问题：** `detail/[id]/page.tsx` 使用了 `useTranslation`、`Button`、`Card` 等组件但未显式导入（依赖 auto-imports）

**修复：**
```typescript
import { Button, Card, Descriptions, Divider, Flex, Table, Tooltip } from 'antd';
import { useTranslation } from 'react-i18next';
```

#### 2. 硬编码中文文本
**问题：** 详情页有 15 处硬编码中文字符串（"部门"、"审核人"、"商品名称" 等）

**修复：** 替换为 i18n 键，例如：
- `'部门'` → `t('page.storage.in.purchase.department')`
- `'审核人'` → `t('page.storage.in.purchase.auditUserName')`
- `'合计'` → `t('page.storage.in.purchase.totalAmount')`

#### 3. i18n 键缺失
**问题：** `page.storage.in.purchase` 仅有 4 个键（add, audit, reverseAudit, title）

**修复：** 扩展到 27 个键，涵盖详情页所有字段：
```typescript
// zh-cn/page.ts & en-us/page.ts
purchase: {
  add, audit, reverseAudit, title,

  createTime, department, expireDate,

  sectionAudit, sectionBasic, sectionBusiness, sectionDetails,
  totalAmount, totalPrice, unitPrice, updateTime
}
```

#### 4. TypeScript 类型定义过窄
**问题：** `src/types/app.d.ts` 中 `page.storage.in.purchase` 类型只包含 4 个字段，导致新增的 i18n 键报 TS2353 错误

**修复：** 同步扩展类型定义到 27 个字段

### 第二轮修复：路由 404 问题

#### 5. 文件路径不符合约定 ⚠️
**问题：** 详情页使用了非标准路径 `detail/[id]/page.tsx`（唯一使用 `page.tsx` 的文件）

**影响：** Vite 的 file-system routing 插件无法识别，导致：
- 路由未生成 → 跳转 404
- 生成的 `src/router/elegant/routes.ts` 中缺少详情路由

**修复：** 重命名为标准路径 `detail/[id].tsx`（对齐其他详情页）

#### 6. 路由元信息预设缺失
**问题：** `build/plugins/router.ts` 的 `ROUTE_META_PRESETS` 中缺少详情页配置

**影响：** 即使路由生成，也无法正确配置 hideInMenu 和 activeMenu

**修复：** 添加两行预设：
```typescript
'(base)_storage_in_purchase_detail': { activeMenu: '/storage/in/purchase', hideInMenu: true },
'(base)_storage_in_purchase_detail_[id]': { activeMenu: '/storage/in/purchase', hideInMenu: true },
```

#### 7. 详情页 handle 导出缺失
**问题：** 详情页组件未导出 `handle` 对象（路由元信息）

**影响：** 无法覆盖默认路由配置

**修复：** 添加 handle 导出：
```typescript
export const handle = {
  hideInMenu: true,
  i18nKey: 'route.(base)_storage_in_purchase',
  keepAlive: false
};
```

#### 8. 路由 i18n 键缺失
**问题：** `src/locales/langs/zh-cn/route.ts` 和 `en-us/route.ts` 中缺少详情页的翻译键

**影响：** TypeScript 报错 TS2322，提示缺少 `"(base)_storage_in_purchase_detail"` 和 `"(base)_storage_in_purchase_detail_[id]"` 键

**修复：** 在两个 route.ts 文件中添加：
```typescript
// zh-cn
'(base)_storage_in_purchase_detail': '采购入库详情',
'(base)_storage_in_purchase_detail_[id]': '采购入库详情',

// en-us
'(base)_storage_in_purchase_detail': 'Purchase Stock In Detail',
'(base)_storage_in_purchase_detail_[id]': 'Purchase Stock In Detail',
```
