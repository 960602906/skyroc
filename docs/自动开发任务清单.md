# SkyRoc 自动开发任务清单

> 执行顺序：从上到下，每次自动化只完成第一个未勾选任务。  
> 完成标准：交付物全部落地，相关测试与 `dotnet build SkyRoc.sln`、`dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj` 通过，文档已同步。仅创建空壳、只写计划或测试失败均不得勾选。

## 执行规则

1. 开始前读取 `AGENTS.md`、`docs/开发进度.md`、本清单及任务对应的 `docs/business-flows/*.md`。
2. 检查 `git status`，保留用户已有改动；领取第一个 `[ ]` 任务，不跳项、不并行领取下一项。
3. 完成实现、迁移、测试和必要文档。任务范围过大时仍只围绕该任务工作，不顺手推进下一项。
4. 验收通过后改为 `[x]`，更新 `docs/开发进度.md` 的断点和日期；测试命令、结果和基线更新 `docs/测试进度.md`；下一项加粗成为唯一当前任务。
5. 如被外部依赖或关键决策阻塞，不勾选、不跳过；实现阻塞写入 `docs/开发进度.md`，测试或环境证据写入 `docs/测试进度.md` 后结束本次运行。
6. 注释属于交付物：新增或修改持久化模型时，必须用 EF Core `HasComment` 为数据库表和字段配置可落库注释并检查迁移；新增或修改仓储层、服务层、控制层时，必须为公共类型和成员补齐说明业务语义的 XML 注释。缺少注释不得勾选任务。
