# SkyRoc 自动业务测试任务清单

> 细则见 `docs/superpowers/specs/2026-07-12-automated-business-testing-design.md`。
> 每次只领取从上到下第一个未勾选 `[ ]` 的**一个业务切片**；失败保留证据，不勾选、不跳项。
> 只用专用 PostgreSQL 测试库；严禁删库/重建/`TRUNCATE`/清空长期数据/改非受管数据。

## 全局规则（摘要）

1. 开工前读 `AGENTS.md`、`docs/开发进度.md`、`docs/测试进度.md`、本清单；`git status` 保留已有改动。
2. 临时数据：单连接事务回滚；跨请求用本轮批次号，结束后按外键逆序精确清理。联调数据按稳定业务键幂等补齐。
3. 数量下限：普通表 ≥20；权限/基础资料 30–50；商品客户等 30–80；订单采购等主表 50–100；明细/流水 100–300。枚举/单例/唯一键表填满合法记录即可。
4. 适用可写字段填有意义值；禁止空串与“测试/暂无/N/A”等占位。金额用 `NumericPrecision`。优先走应用服务/真实 HTTP。
5. 覆盖正常、边界、未认证、权限拒绝、非法状态、幂等、回滚及适用并发；并核对库副作用。
6. 每轮：本切片精确 FQN 测试 + 必要质量/元数据测试 + `dotnet build` + `dotnet format`（以 `docs/自动测试任务提示词.md` 为准；禁止擅自跑全量）。
7. 输出 JSON + Markdown 质量报告。交付物、质量、相关测试、build/format 全过才勾选；测试结果更新 `docs/测试进度.md`，实现断点变化才更新 `docs/开发进度.md`。
