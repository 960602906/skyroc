# SkyRoc 自动业务测试任务清单

> 设计依据：`docs/superpowers/specs/2026-07-12-automated-business-testing-design.md`。
> 执行顺序：每次自动化只领取从上到下第一个未勾选的 `[ ]` 任务；失败时保留证据，不勾选、不跳项。
> 数据安全：只允许使用项目配置的专用 PostgreSQL 测试数据库。该库同时保存长期前端联调数据，严禁删除或重建数据库、全表 `TRUNCATE`、清空长期数据、修改无法确认归属的数据。

## 全局执行规则

1. 每次开始前完整读取 `AGENTS.md`、`docs/开发进度.md`、`docs/自动开发任务清单.md`、本清单、批准设计和全部 `docs/business-flows/*.md`，检查 `git status` 并保留用户已有改动。
2. 自动测试临时数据优先使用事务并回滚；跨 HTTP 请求或跨连接场景使用本轮唯一批次标识，结束后按外键逆序精确清理本轮数据。联调数据使用稳定业务键幂等补齐，禁止模糊名称匹配或删除非本生成器数据。
3. 普通项目表验收下限为 20 条；用户权限和基础资料目标 30–50 条，商品、客户、供应商、采购员、仓库和定价目标 30–80 条，订单、采购、库存、配送、售后和结算主表目标 50–100 条，明细、关系、审计、流水和日志目标 100–300 条。固定枚举、单例配置和受唯一键限制的表填满全部合法记录，不为凑数破坏约束。
4. 生成记录的全部可写且在当前状态下适用的字段必须填写有意义的值；禁止空字符串、纯空白及“测试”“暂无”“N/A”等机械占位值。状态条件字段必须符合业务语义，并至少有一组适用记录覆盖。
5. 优先通过应用服务或真实 HTTP API 形成数据与状态流转；只有没有公开写入口的支撑数据才可使用受控数据库构造器。数量、价格、成本和金额统一使用 `Shared.Constants.NumericPrecision`。
6. 每项测试同时覆盖正常流程、参数边界、未认证、权限拒绝、非法状态、重复请求、幂等、事务回滚和适用的并发场景；除 HTTP 响应外，还要核对主明细、快照、批次、流水、账单、结算余额和关联记录等副作用。
7. 每次运行均须至少执行当前任务相关测试、`dotnet build SkyRoc.sln --no-restore`、`dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --no-build` 和 `dotnet format SkyRoc.sln --verify-no-changes`。需要 PostgreSQL 的测试必须真实执行，不得以 EF Core InMemory 代替。
8. 每次验收输出机器可读和 Markdown 质量报告，至少包含逐表记录数、字段填充率、状态分布、孤儿外键、重复业务编码、临时数据残留和业务一致性检查。只有交付物、数据质量、相关测试、全量构建测试和格式检查全部通过，才能勾选当前任务并更新 `docs/开发进度.md`。

## T0–T14 顺序任务

- [x] **T0 测试基础设施与安全隔离**
  - **目标：** 建立只能连接 SkyRoc 专用 PostgreSQL 测试数据库的测试宿主、迁移入口、事务回滚夹具、批次临时数据登记/清理器和统一报告目录，为后续所有真实数据库测试提供安全边界。
  - **前置条件：** 已批准设计可用；专用测试数据库连接串由测试配置或环境变量提供；数据库名称和环境标识可明确识别为测试用途；不得复用生产库。
  - **数据准备：** 创建最小安全探针记录验证事务回滚；再创建带本轮唯一批次标识的跨连接父子记录验证精确清理。仅操作本轮记录，不生成长期联调数据。
  - **正常场景：** 验证连接目标白名单、待执行迁移应用、单连接事务回滚、跨 HTTP/连接批次登记、按外键逆序清理、重复清理幂等和报告目录创建。
  - **异常场景：** 拒绝非白名单数据库、空连接串、生产环境标识、数据库名不匹配、无批次归属记录、清理范围越界和仍被外键引用的数据；模拟业务异常并确认事务回滚；模拟清理失败并保留证据。
  - **数据库断言：** 回滚后探针记录为零；清理前只新增本轮批次记录；清理后本轮临时记录为零且既有长期记录主键、更新时间和数量不变；迁移历史与 EF Core 模型一致；禁止产生 `DROP DATABASE`、`TRUNCATE` 或无归属批量删除。
  - **交付物：** PostgreSQL 测试配置约定、测试宿主/集合夹具、数据库安全守卫、事务作用域、批次上下文与清理器、报告输出目录、基础设施测试和中文使用说明。
  - **验证命令：** `dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~PostgreSql|FullyQualifiedName~DatabaseSafety|FullyQualifiedName~BatchCleanup"`；`dotnet ef database update --project Infrastructure --startup-project SkyRoc`；以及全局规则第 7 条命令。
  - **勾选条件：** 真实 PostgreSQL 下安全守卫、回滚和精确清理测试全部通过；错误目标均被 fail-closed 拒绝；无本轮临时残留；基础质量报告成功生成；全量验证通过。

- [ ] **T1 全表盘点与数据质量规则**
  - **目标：** 自动比对 EF Core 运行时模型与 PostgreSQL 目录，形成业务表、列、主外键、唯一约束、可空性、状态字段、数量级和字段适用性规则的唯一清单及统一质量验收器。
  - **前置条件：** T0 已完成；专用测试数据库已应用最新迁移；安全守卫和只读元数据连接可用。
  - **数据准备：** 只读采集 EF 模型和 `pg_catalog`/`information_schema`；集中登记固定枚举、单例配置、唯一键限制表和条件字段例外及理由，不改动长期数据。
  - **正常场景：** 自动发现全部项目表和持久化列；排除 `__EFMigrationsHistory`；核对表列、键、精度、默认值、注释和删除行为；按表分类计算目标数量；生成字段适用条件与状态语义规则。
  - **异常场景：** 检出 EF/数据库缺表缺列、迁移漂移、无理由例外、重复登记、未知状态字段、硬编码数值精度、新增表未分类或新增可空字段未定义适用性时失败。
  - **数据库断言：** 每个 EF 实体表在 PostgreSQL 唯一对应；数据库业务表无未登记孤表；外键、唯一约束和 decimal 精度与模型一致；全部表列有注释；例外清单只包含约束可证明的记录。
  - **交付物：** 机器可读元数据清单、Markdown 表盘点、固定配置例外清单、字段适用条件清单、统一质量验收器及回归测试。
  - **验证命令：** `dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~Metadata|FullyQualifiedName~DataQuality|FullyQualifiedName~DatabaseDocumentation"`；真实 PostgreSQL 盘点/报告命令；以及全局规则第 7 条命令。
  - **勾选条件：** EF 与 PostgreSQL 全表全列互相覆盖；所有例外有可验证理由；质量验收器能用故障样本稳定失败并在正常库生成两种报告；全量验证通过。

- [ ] **T2 全量联调数据生成器**
  - **目标：** 按依赖顺序幂等生成覆盖全部项目表、业务状态和适用字段的长期前端联调数据，并达到分级数量目标。
  - **前置条件：** T1 已完成；表分类、字段规则和安全守卫可用；专用数据库可供自动测试与前端联调共同使用。
  - **数据准备：** 使用稳定前缀和稳定业务键先生成权限/基础资料，再经应用服务或真实 HTTP 构造订单、采购、库存、配送、售后、财务、溯源、打印、公告、审计链路；无公开写入口的数据使用受控构造器。
  - **正常场景：** 首次补齐、二次运行不重复、缺失记录恢复、可维护字段更新、跨日期/区域/金额/状态多样性、主链路与逆向链路完整生成；全部适用可写字段使用有意义值。
  - **异常场景：** 冲突稳定键、外部人工同名数据、目标数量受约束不可达、状态字段不适用、生成中途失败和重试；失败时停止后续阶段且保留既有长期数据。
  - **数据库断言：** 普通表不少于 20 条并达到设计目标区间；受约束表填满合法记录；适用字段填充率 100%；无空白/机械占位、孤儿外键、重复业务编码、非法状态、负库存和账实不一致；重复运行业务键与数量稳定。
  - **交付物：** 幂等数据生成命令、分层生成器、稳定键目录、受控构造器、联调账号说明、机器可读与 Markdown 数据质量报告及生成器测试。
  - **验证命令：** 联调数据生成命令连续执行两次；`dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~DemoData|FullyQualifiedName~DataQuality"`；以及全局规则第 7 条命令。
  - **勾选条件：** 两次生成均成功且第二次无重复增量；数量、填充率、状态覆盖和一致性全部达标；不修改非生成器管理数据；无临时残留；全量验证通过。

- [ ] **T3 认证、个人中心与系统权限**
  - **目标：** 验证登录、刷新、注销、本人资料、修改密码、用户/角色/菜单/按钮/部门权限、动态路由、令牌失效和审计闭环。
  - **前置条件：** T2 已完成；联调用户、角色、菜单、按钮、部门和授权关系齐全；测试账号密码仅来自安全测试配置。
  - **数据准备：** 幂等保留 30–50 个长期用户/角色/菜单/部门组合；每个测试使用独立账号或本轮批次授权关系，密码变化场景在事务回滚或精确恢复范围内执行。
  - **正常场景：** 登录、刷新令牌、查询信息/路由、更新本人资料、修改密码后重新登录、注销、角色菜单分配、用户角色分配、通配权限和最小权限访问。
  - **异常场景：** 错误/边界凭据、禁用用户、过期/伪造/已注销令牌、旧密码错误、越权修改他人、相邻权限越权、重复刷新/注销、并发角色或密码变更、业务失败回滚。
  - **数据库断言：** 用户密码仅存哈希；资料和密码只影响本人；角色菜单及用户角色关系准确；刷新/注销后的缓存令牌状态正确；登录日志和脱敏操作日志完整且无密码/令牌泄漏；失败操作无脏写。
  - **交付物：** 真实 PostgreSQL 服务/API 测试、HTTP 授权矩阵、令牌生命周期与并发测试、相关质量报告和必要缺陷修复。
  - **验证命令：** `dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~Auth|FullyQualifiedName~Authorization|FullyQualifiedName~Profile|FullyQualifiedName~System"`；以及全局规则第 7 条命令。
  - **勾选条件：** 所有受保护接口至少覆盖允许/无权限/未认证；令牌与密码副作用和审计一致；无敏感信息泄漏或临时残留；质量与全量验证通过。

- [ ] **T4 商品、客户、供应商与定价**
  - **目标：** 验证商品分类、商品/单位、公司/客户/标签/子账号、供应商、采购员、仓库、报价、协议价和采购规则的 CRUD、启停、审核及价格优先级。
  - **前置条件：** T3 已完成；基础资料权限矩阵和联调身份可用；各类基础资料长期数据达到目标数量。
  - **数据准备：** 稳定生成 30–80 条商品、客户、供应商、采购员、仓库和定价数据及 100–300 条关系明细，覆盖启用/停用、审核/未审核、有效/过期协议和多个单位换算。
  - **正常场景：** CRUD、批量删除/状态、引用保护、分类/标签树、单位换算、报价商品维护与审核、客户绑定、协议价有效期和客户商品价格选择、采购规则适用关系。
  - **异常场景：** 重复编码/名称、空白和超长字段、非法税率/精度/日期、删除被引用资料、停用资料继续建单、重复绑定、协议价冲突、无权限/未认证、并发创建唯一键和事务回滚。
  - **数据库断言：** 主关系表数量及外键正确；快照来源和优先级结果准确；唯一约束生效；删除/停用不破坏历史引用；失败和并发冲突无部分关系残留。
  - **交付物：** 基础资料与定价真实 PostgreSQL/API 测试、价格优先级用例、数据质量报告和必要缺陷修复。
  - **验证命令：** `dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~Goods|FullyQualifiedName~Customer|FullyQualifiedName~Supplier|FullyQualifiedName~Pricing|FullyQualifiedName~PurchaseRule|FullyQualifiedName~Ware"`；以及全局规则第 7 条命令。
  - **勾选条件：** CRUD、关系、状态、价格来源、边界、权限、幂等和并发均通过；数据库副作用完整；质量与全量验证通过。

- [ ] **T5 销售订单**
  - **目标：** 验证销售订单 CRUD、金额/单位计算、业务快照、提交/通过/驳回/重提状态机、并发与幂等。
  - **前置条件：** T4 已完成；有效客户、报价/协议价、商品单位、仓库和权限数据可用。
  - **数据准备：** 生成 50–100 张多日期、多客户、多商品订单及 100–300 条明细/审核记录，覆盖草稿、待审核、驳回、待分拣、配送中、签收等可保留状态。
  - **正常场景：** 分页/详情、创建/编辑/删除草稿、价格选择、数量换算、金额舍入、提交、审核通过、驳回、修改后重提、按商品查询和重复读取。
  - **异常场景：** 空明细、重复商品、极小/大数量、负值和精度边界、无效单位/客户/报价、非法状态编辑删除/流转、重复审核/重提、权限拒绝、并发审核与编辑、事务失败。
  - **数据库断言：** 订单与明细金额按 `NumericPrecision`；客户/商品/单位/价格/地址快照正确；审核记录只追加；状态与操作人/时间/意见一致；失败操作不改主明细且无孤儿。
  - **交付物：** 真实 PostgreSQL 服务/API/并发测试、订单数据集与质量报告、必要缺陷修复。
  - **验证命令：** `dotnet test SkyRoc.Tests/SkyRoc.Tests.csproj --filter "FullyQualifiedName~OrderServiceTests|FullyQualifiedName~OrderApiIntegrationTests|FullyQualifiedName~SaleOrder"`；以及全局规则第 7 条命令。
  - **勾选条件：** 正常与异常状态机、金额、快照、权限、并发、幂等和回滚全部通过；无批次残留；质量与全量验证通过。

- [ ] **T6 采购计划与采购单**
  - **目标：** 验证订单生成采购计划、手工计划、分配、合并拆分、计划数量守恒、采购单生成/维护/完成/取消及来源回写。
  - **前置条件：** T5 已完成；已审核订单、商品采购单位、供应商、采购员和采购规则可用。
  - **数据准备：** 生成 50–100 张计划/采购单和 100–300 条明细/来源关系，覆盖直供/自采、未发布/部分/全部生成、草稿/完成/取消状态。
  - **正常场景：** 按订单幂等生成计划、手工新增、供应商/采购员分配与清除、兼容计划合并、按订单和数量拆分、生成采购单、编辑、删除、完成和取消。
  - **异常场景：** 未审核或已生成订单、重复生成、来源数量超限、不同单位/模式错误合并、拆分不守恒、已占用计划修改、非法状态采购单、无权限、并发生成/取消和中途回滚。
  - **数据库断言：** 计划需求、拆分、合并和已采购数量守恒；订单采购标记、来源关系、供应商/采购员/商品/单位快照正确；取消/删除原子回写占用；失败无半张单或残留关系。

- [ ] **T7 入库、出库、库存和盘点**
  - **目标：** 验证采购/其他/销售退货入库、销售/采购退货/其他出库、移动加权成本、库存锁定、反审核、批次台账和盘点调整。
  - **前置条件：** T6 已完成；采购单、订单、售后来源、仓库、商品单位和库存权限数据可用。
  - **数据准备：** 生成 50–100 张各类库存主单和盘点单、100–300 条明细/批次/流水，覆盖多仓、多批次、效期、成本、预留、盘盈盘亏及审核/反审核状态。
