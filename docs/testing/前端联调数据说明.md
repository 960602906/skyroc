# 前端联调数据说明

> 与自动业务测试共用专用 PostgreSQL 库 `skyroc`。本文只说明联调可用边界与稳定键约定，**不记录密码、连接串或任何密钥**。

## 数据库与安全边界

- 专用库名必须为 `skyroc`（见 `SkyRoc.Tests/postgresql-testsettings.json`）。
- 禁止删库、重建、全表 `TRUNCATE`、清空长期数据或修改无法确认归属的非受管记录。
- 自动测试临时数据前缀为 `SKYROC-AUTOTEST-*`；结束后按批次精确清理，不得残留。
- 长期联调数据前缀为 `SKYROC-DEMO-*`，由生成器按完整稳定键幂等补齐/更新。

## 稳定业务键

生成器通过 `DemoDataStableKeyCatalog.Create("<领域>", 序号)` 得到完整键，形如：

| 领域示例 | 示例键 | 用途 |
| --- | --- | --- |
| `GOODS` | `SKYROC-DEMO-GOODS-001` | 商品 |
| `CUSTOMER` | `SKYROC-DEMO-CUSTOMER-001` | 客户 |
| `SUPPLIER` | `SKYROC-DEMO-SUPPLIER-001` | 供应商 |
| `WARE` | `SKYROC-DEMO-WARE-001` | 仓库 |
| `SYSTEM-USER` | `SKYROC-DEMO-SYSTEM-USER-001` | 联调系统用户登录名 |
| `SYSTEM-ROLE` | `SKYROC-DEMO-SYSTEM-ROLE-001` | 联调角色编码 |

查询与幂等更新必须使用**完整键**精确匹配，禁止用名称模糊定位。

## 账号来源（不写密码）

| 类型 | 标识来源 | 密码来源 |
| --- | --- | --- |
| 开发管理员 | `DevSeed` 节中的管理员用户名（默认 `admin`，仅本地受控种子） | 仅本地环境变量或未入库的 DevSeed 管理员口令配置 |
| 联调系统用户 | `SKYROC-DEMO-SYSTEM-USER-001` … `030` | 由生成器确定性派生，**不入仓**；向本地维护者索取或本地重置 |
| 客户子账号 | 生成器为受管客户补齐的稳定用户名 | 同上，不入仓 |

联调前确认：专用库已 `dotnet ef database update`，且 `FullAcceptancePostgreSqlTests` / 最新质量报告中 `demoDataAcceptance.isReady=true`。

## 幂等补齐与失败恢复

1. 通过测试宿主或受控入口调用 `GenerateDemoDataAsync`（需已存在可审计系统用户）。
2. 连续两次生成应成功；第二次不得产生重复业务键增量。
3. 生成失败时**停止**后续破坏性操作；修复后按稳定键继续幂等补齐，不删除无法确认归属的数据。
4. 菜单按钮等受管字段若漂移，先按稳定键核对再修复生成器，不得扩大质量例外清单绕过。

## 场景索引（按稳定键追踪）

主正向：销售订单 → 采购计划/采购单 → 采购入库 → 销售出库 → 配送签收 → 客户结款。  
逆向：售后/取货/退货入库/应收冲减；采购退货/应付调整。  
支撑：检测溯源、报表驾驶舱、导入导出、打印、运营公告与审计。

具体单据编号仍以库内 `SKYROC-DEMO-*` / 业务单号为准，前端可用编码或名称中的联调字样检索。

## 一键验收入口

```powershell
./scripts/Invoke-SkyRocFullAcceptance.ps1
```

默认执行：白名单校验 → 迁移 → `has-pending-model-changes` → `FullAcceptancePostgreSqlTests` 与质量/元数据精确门禁 → `dotnet format --verify-no-changes`。  
完整 `SkyRoc.Tests` 仅在人工明确要求时加 `-IncludeFullTestSuite`。细节见 [PostgreSQL自动业务测试.md](./PostgreSQL自动业务测试.md)。
