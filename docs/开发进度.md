## 续接规则

每次开发开始先读取本文件、[自动开发任务清单](./自动开发任务清单.md) 和对应的 `docs/business-flows/*.md`，检查 `git status`，再运行构建与测试。每次只领取清单中第一个未完成任务。只有交付物、测试和文档均通过验收才能勾选；结束时更新断点、模块表、核对日期和测试基线。
