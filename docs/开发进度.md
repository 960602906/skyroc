已经完成到：分层架构、JWT 登录/刷新/注销、动态路由、用户/角色/菜单/按钮/部门、个人中心与修改密码，以及商品、客户、供应商、采购员、仓库、报价、协议价、采购规则的数据库模型和基础 API；权限基础设施已具备稳定权限码、JWT 权限声明、授权要求和处理器，系统管理及商品、客户、定价、采购、仓库基础资料接口均已接入细粒度授权；第一阶段基础资料已有服务级回归测试，Swagger 能区分匿名/受保护操作并展示权限码；订单阶段已新增销售订单、订单明细、审核记录实体、EF 配置和可回滚迁移，补齐订单/明细/审核 DTO、双查询模型、映射、验证和仓储，并已实现事务化订单分页、详情、创建、编辑、删除以及提交、通过、驳回、重提状态机；订单接口现已接入读、建、改、删、审核五类细粒度权限，Swagger 展示对应权限码，HTTP 脚本记录了创建到审核、删除的主流程，API 集成测试覆盖认证、授权、路由、序列化和完整状态流转；采购阶段已新增采购计划、计划明细和订单来源关联实体、EF 配置及可回滚迁移，模型支持采购模式、采购单生成状态、供应商/采购员快照、商品/采购单位快照和合并拆分所需的来源数量关系。当前共记录 182 个 HTTP 请求。

每次开发开始先读取本文件、[自动开发任务清单](./自动开发任务清单.md) 和对应的 `docs/business-flows/*.md`，检查 `git status`，再运行构建与测试。每次只领取清单中第一个未完成任务。只有交付物、测试和文档均通过验收才能勾选；结束时更新断点、模块表、核对日期和测试基线。
