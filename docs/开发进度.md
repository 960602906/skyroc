## 最近开发记录

### 2026-07-23 AI/MCP 能力契约与六表持久化

- P6-01A 新增能力 Registry/Executor/ResponseProjector 抽象、厂商无关 operationId/Schema/调用结果、`[AiOperation]`/`[AiIgnore]`、GET/HEAD 读取与写方法确认默认值，以及 `ai:action-draft:confirm` 权限；真正的 ApiExplorer 注册和 HTTP 执行仍留在 P6-07。
- `Ai:CapabilityGateway` 固定内部来源、能力搜索 20 项、结果 65536 字节、委托令牌 60 秒和高风险确认 300 秒边界。按用户当前决定，DeepSeek 与 GLM 的开发/测试 Key 直接保存在已跟踪的 `SkyRoc/appsettings.json`；不再加载额外本地配置文件，同时保留对应环境变量名称用于回退或后续迁移。
- P6-02 新增 `AiConversation`、`AiMessage`、`AiActionDraft`、`AiOrderDraft`、`AiOrderDraftDetail`、`McpAccessToken` 六张表；全部实体、字段、关系、索引、约束和迁移均带中文注释。
- 通用操作草稿将参数规范化后，以“用户 + operationId + 参数”计算 SHA-256 哈希；确认时任一项变化即拒绝。两类草稿均有用户级幂等索引和 30 分钟有效期，会话消息使用唯一序号游标。
- 旧五表 WIP 迁移已从专用数据库安全回退，历史测试残留使用项目现有精确清理器处理；新版六表迁移通过实际 Up/Down/Up 验证后，旧 `stash@{0}` 已删除。

### 2026-07-23 AI/MCP 全量 API 能力范围修订

- 用户明确要求“只要是后端提供的业务接口，AI 都可以调用”；详细计划已从固定业务 MCP 工具改为“能力搜索 → 单 operationId Schema → 受控调用原 REST API”。
- P6-00、P6-01 的代码和测试结论保持有效，不回退完成状态；新增 P6-01A 补充能力 Registry/Executor/ResponseProjector、`[AiOperation]`/`[AiIgnore]`、风险确认模型和网关配置。
- P6-01A 成为唯一下一任务；其测试未通过前不得进入 P6-02。写操作默认生成通用草稿，高风险操作二次确认，销售订单继续走专用价格复验草稿。
- 外部 MCP 使用同一能力目录，但受 PAT scope 与用户实时权限交集限制；模型不能控制 URL、HTTP 方法、请求头、用户或权限。

### 2026-07-22 AI/MCP 公共契约、配置和权限骨架

- 新增厂商无关的对话消息、工具、流分片、用量、能力和安全错误模型，以及 `IAiModelProvider` / `IAiModelProviderRegistry`；活动 Registry 只按 `Ai:ActiveProvider` 选择一个 Adapter，并在启用时拒绝未知 Provider、未注册或重复 Adapter。
- 新增 `AiOptions`、`AiProviderOptions`、`McpOptions` 与宿主启动校验；地址、模型、数值边界和密钥环境变量缺失均给出可定位配置路径，AI 或外部 MCP 关闭时不会读取对应密钥，也不会构造模型适配器。
- `appsettings.json` 最初只保存环境变量名称；按用户 2026-07-23 后续决定，DeepSeek 与 GLM 改为直接保存开发/测试 Key，同时保留环境变量回退能力。新增 `ai:assistant:use`、`ai:order-draft:create`、`ai:mcp-token:manage` 三项稳定权限并纳入完整性测试。
- P6-01 已在自动开发任务清单中勾选；当时下一任务为 P6-02，2026-07-23 范围修订后由 P6-01A 插入承接能力网关增量。命令与验证结果见 [`测试进度.md`](./测试进度.md)。

### 2026-07-22 前端表格短请求加载抖动修复

- 通用 `useTable` 将 Ant Table 的加载遮罩延迟 200ms：短请求不再触发表格透明度动画，长请求仍保留旋转图标和遮罩反馈。
- `TableHeaderOperation` 同时兼容布尔值和 Ant Design `SpinProps`，刷新按钮的旋转状态继续与真实请求状态同步；售后单、取货任务及其他复用分页表格统一生效。

### 2026-07-21 下拉选项分级与限量搜索治理

- 新增统一 `SelectionOptionDto`、无总数统计的 `SelectionOptionSearchResultDto`，以及 `/options/search`、`/options/resolve`、`/options/bounded`；增长型数据默认 20、最多 50，解析最多 100，有界数据超过 500 返回业务校验失败。
- 通用基础资料在数据库侧 `AsNoTracking` 后直接投影；销售订单单独投影“订单号 + 客户名称”，空关键词按最近创建时间排序。旧全量 GET 与报价/协议旧 `/options` 保留一个发布周期并在 Swagger 标记弃用。
- 迁移 `20260721024046_AddSelectionOptionSearchIndexes` 启用 `pg_trgm`，为商品、客户、供应商、报价、协议和销售订单建立 `lower(...)` GIN trigram 索引；Down 只删除本次索引，不删除共享扩展。
- 前端新增 `RemoteOptionSelect`/`useRemoteOptions`，完成客户、商品、供应商、报价、协议和销售订单选择迁移；公司、仓库、标签、采购员、司机、角色改走有界轻量接口，商品单位按商品联动，写操作会失效增长型资源选项缓存。

### 2026-07-20 打印模板启用态 bool 哨兵修复

- `PrintTemplateConfiguration`：移除 `is_enabled` 的 `HasDefaultValue(true)+HasSentinel(false)`，避免显式写入 `false`（停用）被 EF 当作未设置而落到数据库默认值 `true`。
- 迁移 `20260720142527_FixPrintTemplateIsEnabledBoolSentinel`：去掉 `print_template.is_enabled` 列默认值；部署需先 `dotnet ef database update`。

### 2026-07-20 商品上下架 bool 哨兵修复

- `GoodsConfiguration` / `QuotationGoodsConfiguration`：移除 `is_on_sale` 的 `HasDefaultValue(true)+HasSentinel(false)`，避免显式写入 `false`（下架）被 EF 当作未设置而落到数据库默认值 `true`。
- 迁移 `20260720140536_FixGoodsIsOnSaleBoolSentinel`：去掉 `goods`、`quotation_goods.is_on_sale` 列默认值；部署需先 `dotnet ef database update`。

### 2026-07-20 检测报告更新唯一索引冲突修复

- `TraceabilityService.UpdateInspectionReportAsync`：商品明细与附件改为按入库明细主键/展示顺序就地同步，避免 Clear+Add 在 `(inspection_report_id, stock_in_detail_id)` 唯一索引上同批 INSERT/DELETE 冲突导致更新失败。

### 2026-07-20 库存并发扣减锁修复与测试库切换

- 专用测试库白名单由 `neondb` 调整为 `skyroc`；测试连接优先读取 `SkyRoc/appsettings.Development.json`。
- `StockBatchRepository.GetByIdentityForUpdateAsync`：在执行 `FOR UPDATE` 前分离已跟踪的同标识批次，避免出库单 `ThenInclude(StockBatch)` 的旧快照吞掉行锁重读结果，修复并发超额出库双胜写坏库存。

### 2026-07-20 售后列表与前端续接

- 售后分页已收敛为总数查询加单条轻量投影；列表 DTO 与详情 DTO 分离，按创建时间和主键倒序分页。
