# SkyRoc 前端菜单分布规划

> 用途：前后端联调时的侧边栏/动态路由约定。  
> 原则：顶层按业务中心；基础资料与系统能力再分子组；**任意目录节点至少挂 2 个子项**（避免单叶子悬空）。  
> 依据：`docs/business-flows/99-main-business-flow.md`、现有控制器路由与 `PermissionCodes`。  
> 核对日期：2026-07-15

## 1. 约定

| 项 | 说明 |
| --- | --- |
| 菜单来源 | 后端 `sys_menu` + `getRoutes`；本文是业务侧推荐结构，落地时写入菜单表/种子即可 |
| 路由风格 | 建议前端 path 使用下文「建议 path」；可按脚手架微调，但 **name / 权限码** 建议保持稳定 |
| 权限 | 叶子菜单绑定资源读权限；按钮级动作（增删改审）走 `sys_menu_button`，编码见 `PermissionCodes` |
| 隐藏页 | 详情/编辑/新增表单建议 `hideInMenu=true`，用 `activeMenu` 高亮所属列表 |
| 个人中心 | `/user-center` 不进侧栏，登录后顶栏入口即可 |
| 现有种子 | `DbSeeder` 里仍是模板演示菜单，**不代表业务菜单**；对接前需按本文替换或增量写入 |

## 3. 分组明细（对接表）

说明：

- **类型**：`目录` 仅展开；`菜单` 对应页面。
- **读权限**：进入页面至少需要的权限；无权限则 `getRoutes` 不返回该节点。
- **主 API**：列表/主操作入口，细节以 Swagger 为准。

## 4. 侧栏排序建议

自上而下：

1. 首页看板  
2. 订单中心（含售后、取货）  
3. 采购中心  
4. 库存中心  
5. 配送中心  
6. 财务中心  
7. 溯源中心  
8. 报表分析  
9. 基础资料  
10. 系统管理  

与主业务链路「下单 → 采购 → 出入库 → 配送 → 售后 → 结算」一致，资料与系统沉底。

## 5. 按钮权限（页面内）

叶子菜单只解决「能不能进」。页内按钮按模块挂常见动作，编码格式 `module:resource:action`：

| 模块 | 常见按钮权限 |
| --- | --- |
| 订单 | `business:order:create/update/delete/audit` |
| 采购 | `business:purchase:create/update/delete` |
| 库存 | `business:storage:create/update/delete`（审核/反审核归 update） |
| 配送 | `business:delivery:create/update/delete` |
| 售后 | `business:after-sales:create/update/delete/audit` |
| 财务 | `business:finance:create/update/delete` |
| 溯源 | `business:traceability:create/update/delete` |
| 定价 | `business:pricing:create/update/delete/audit` |
| 系统用户 | `system:user:*`、`system:user:assign-roles` |
| 系统角色 | `system:role:*`、`system:role:assign-menus` |

完整清单见 `Shared/Constants/PermissionCodes.cs`。

## 6. 联调检查清单

- [ ] 每个目录节点 ≥ 2 个子项（含二级目录）
- [ ] 角色只授叶子菜单时，父目录仍能展开（`getRoutes` 需带上祖先；若后端只返回叶子，前端需补父节点）
- [ ] admin（`*:*:*`）可见全树；业务角色按读权限裁剪
- [ ] 详情/编辑页不进侧栏，但刷新后仍高亮列表菜单
- [ ] 报价商品、协议价商品、菜单按钮不单独占侧栏
- [ ] 公开溯源二维码页不进后台菜单

## 7. 与后端种子的关系

当前 `Infrastructure/Data/DbSeeder.cs` 仍为前端脚手架演示结构（`manage` / `function` / `document` 等）。正式对接时建议：
