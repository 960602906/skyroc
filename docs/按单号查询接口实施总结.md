## 实施模式总结

每个单据需要修改以下 5 层（模式完全一致）：

1. **Repository 接口** (`Domain/Interfaces/*/I*Repository.cs`)
   ```csharp
   Task<Entity?> GetBy[No]Async(string no);
   ```

2. **Repository 实现** (`Infrastructure/Repositories/*Repository.cs`)
   ```csharp
   public async Task<Entity?> GetBy[No]Async(string no)
   {
       var normalizedNo = no.Trim();
       return await BuildDetailQuery()
           .FirstOrDefaultAsync(x => x.[NoField] == normalizedNo);
   }
   ```

3. **Service 接口** (`Application/Interfaces/*/I*Service.cs`)
   ```csharp
   Task<EntityDto> GetBy[No]Async(string no);
   ```

4. **Service 实现** (`Application/Services/*Service.cs`)
   ```csharp
   public async Task<EntityDto> GetBy[No]Async(string no)
   {
       if (string.IsNullOrWhiteSpace(no))
           throw new BusinessException("[类型]单号不能为空");
       
       var entity = await repository.GetBy[No]Async(no.Trim());
       if (entity == null)
           throw new BusinessException($"[类型]单号 {no} 不存在");
       
       return mapper.Map<EntityDto>(entity);
   }
   ```

5. **Controller** (`SkyRoc/Controllers/*Controller.cs`)
   ```csharp
   [HttpGet("by-no/{no}")]
   [ResourcePermission(PermissionActions.Read)]
   public async Task<ActionResult<ApiResponse<Dto>>> GetBy[No](string no)
   {
       var result = await service.GetBy[No]Async(no);
       return Ok(ApiResponse<Dto>.Ok(result));
   }
   ```

## 技术细节

### 查询行为
- 单号自动 `Trim()` 去除前后空格
- 使用 `BuildDetailQuery()` 预加载完整聚合（与 GetById 一致）
- 不存在时抛出 `BusinessException`（符合项目惯例）
- HTTP 状态码统一返回 `200`，业务结果在 `ApiResponse.code` 中

### 路由设计
- 路由模式: `GET /api/{resource}/by-no/{no}`
- 与 `GET /api/{resource}/{id:guid}` 互不冲突
- 前端调用示例: `GET /api/orders/by-no/SO-2024-0001`

### 性能考虑
- 单号字段已有数据库索引（`ExistsNoAsync` 已在用）
- 查询性能与 GetById 相当
- 对于入库/出库，增加了业务类型校验，防止跨类型混淆
