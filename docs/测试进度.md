## 测试记录规范

每次测试只在本文追加一条简洁记录，包含：

1. 日期、任务/业务切片和精确测试类或 FQN。
2. 实际执行命令、通过/失败数量和耗时。
3. 数据质量报告名称、临时数据清理结果和重要副作用断言。
4. 环境阻塞或已知失败的证据；失败任务不得勾选。
5. 下一项唯一测试任务或当前任务的下一切片。

不要在本文重复业务实现过程、Controller/DTO 设计或前端页面信息；这些内容更新到 [`开发进度.md`](./开发进度.md) 或 `client/docs/` 的前端进度文档。

## 测试续接规则

1. 开始前读取本文件、[`自动测试任务清单.md`](./自动测试任务清单.md)、`AGENTS.md` 和相关业务流程文档；检查 `git status`。
2. 若已有 `dotnet test`、testhost 或 vstest 进程，本轮跳过，不终止现有进程。
3. 严格遵守自动测试任务清单的安全边界、数据准备、精确清理和命令范围。
4. 业务测试完成后先更新本文的结果/基线；若同时改变了实现状态，再单独更新 [`开发进度.md`](./开发进度.md)。
